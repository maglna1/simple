# 示例 00：凭据管理

这是整个学习合集的**前置知识**。本示例不调用任何 LLM，仅演示：

1. `.env` 文件是否被 `python-dotenv` 正确加载
2. 三个必需环境变量（`MINIMAX_API_KEY` / `MINIMAX_BASE_URL` / `MINIMAX_MODEL`）的就绪状态
3. 缺失时如何给出可操作的错误提示

## 运行

在仓库根目录执行：

```bash
python examples/00_credentials/main.py
```

```
  ✓  MINIMAX_MODEL: <set, len=11>
------------------------------------------------------------
✗ 缺少必需的环境变量。请按以下步骤修复：
    1. 在仓库根目录执行：cp .env.example .env
    2. 编辑 .env，把占位符替换为真实值
    3. 重新运行本示例
```

## .env 加载原理

1. `utils/llm.py` 顶部调用 `load_dotenv()`，会从当前工作目录向上搜索 `.env` 文件。
2. 当你从 `examples/00_credentials/` 运行 `python main.py` 时，当前工作目录就是那个示例目录；`python-dotenv` 会向上找到仓库根目录的 `.env`。
3. 加载后，`.env` 中的键值对被合并进 `os.environ`。
4. **优先级**：已有的真实环境变量 > `.env` 中的值。这让 CI / Docker 场景下可以直接用真实环境变量覆盖 `.env`。

## 安全准则

1. **永不把 `.env` 提交到 git**。仓库的 `.gitignore` 已经包含 `.env`，请确认它生效（`git check-ignore -v .env` 应命中规则）。
2. **永不把真实 key 贴到任何聊天工具、issue、截图**。一旦泄露，立即到 MiniMax 控制台 rotate。
3. **不要在代码里硬编码 key**。本仓库所有示例都通过 `utils/llm.py` 的工厂函数读取环境变量。
4. **示例 00 只打印 key 长度，不打印明文**——这是示例代码本身的安全示范。
